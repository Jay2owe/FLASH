# Stage 05: Bin Macro Scan Async

## Why

`BinConfigIO` reads `.bin/Channel_Data.txt` / `FLASH/Set Up Configuration/.settings/Channel_Data.txt` during dialog open. The small text reads are unavoidable, but saved custom filter preset discovery still calls `File.listFiles()` on every custom-filter preset directory (`src/main/java/flash/pipeline/bin/BinConfigIO.java:474-485`, `src/main/java/flash/pipeline/bin/BinConfigIO.java:479`). Those directories can live on Dropbox or a slow network drive, so the read can pause for 50-500 ms.

Be precise about the current behavior before changing it: `readFromDirectory` already avoids a full legacy inference scan when every channel has a stored filter token (`src/main/java/flash/pipeline/bin/BinConfigIO.java:65-71`), and `resolveFilterPresetForChannel` only calls `lazyInferred` for missing, generic `Custom`, or stale stored presets (`src/main/java/flash/pipeline/bin/BinConfigIO.java:422-452`, `src/main/java/flash/pipeline/bin/BinConfigIO.java:462-471`). It still scans synchronously for old or short configs through `inferFilterPresets` (`src/main/java/flash/pipeline/bin/BinConfigIO.java:388-394`) and when a channel macro must be matched against saved custom presets (`src/main/java/flash/pipeline/bin/BinConfigIO.java:397-419`).

The user-visible dropdown stall is not owned by `BinConfigIO` alone. `CreateBinFileAnalysis.filterPresetOptions` synchronously calls its own duplicate preset-folder scan before the filter dialogs are shown (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2381-2395`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2439-2452`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2444`). The dropdown can render with bundled presets plus the current saved selection, then merge saved custom preset names after paint.

## Prerequisites

- Read `docs/fast-ui-opening/00_overview.md`.
- Keep Java 8 compatibility: do not use newer `CompletableFuture` helpers such as `orTimeout`; use Java 8 APIs only (`src/main/java/flash/pipeline/bin/BinConfigIO.java:23`).

## Scope

- Extract the saved custom filter preset directory scan into one shared helper, either `BinMacroIndex` in `src/main/java/flash/pipeline/bin/` or a package-private helper beside `BinConfigIO`. It must scan the directories from `FlashProjectLayout.customFilterPresetReadDirs()` (`src/main/java/flash/pipeline/io/FlashProjectLayout.java:425-428`) and preserve existing case-insensitive de-duplication / sorting semantics from `CreateBinFileAnalysis.addUniqueOption` and `listSavedCustomFilterPresets` (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2397-2404`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2439-2452`).
- Keep `BinConfigIO`'s config-read semantics synchronous for resolved channel values. Headless and downstream analyses expect `channelFilterPresets` to be populated immediately after `readFromDirectory` / `readPartialFromDirectory` (`src/main/java/flash/pipeline/bin/BinConfigIO.java:45-83`, `src/main/java/flash/pipeline/bin/BinConfigIO.java:123-164`). Do not replace that with a public UI-oriented blocking accessor unless the implementation makes it unavoidable.
- Add an async option-list path for UI dropdowns: base options should be bundled presets from `NamedFilterLoader.FILTER_NAMES` plus `Custom` and the current selected preset (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:153-164`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2381-2395`). Saved custom preset names arrive later from the shared helper.
- Change `CreateBinFileAnalysis.showFilteredFilterPresetsPage` and the selective override dialog so each `JComboBox` paints immediately, then merges async saved custom preset names on the EDT (event dispatch thread) (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:533-556`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:1115-1132`, `src/main/java/flash/pipeline/ui/PipelineDialog.java:595-608`).
- Change the embedded filter stage path so `createFilterParameterStage` no longer blocks on `filterPresetOptionList` before `ConfigQcDialog` is constructed (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:6217-6257`, `src/main/java/flash/pipeline/ui/config/ConfigQcDialog.java:373-381`). `FilterParameterStage` will need a small async refresh hook or callback because it currently freezes `presetOptions` in the constructor (`src/main/java/flash/pipeline/ui/config/FilterParameterStage.java:195-224`).
- Dropdown behavior:
  1. Initial model contains bundled presets, `Custom`, and the current saved/selected preset; never make `"Loading filters..."` the selected value (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2390-2394`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:552-553`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:1129-1131`).
  2. Optional `"Loading filters..."` placeholder may be present but must be disabled / ignored by save paths (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2236-2237`, `src/main/java/flash/pipeline/ui/config/FilterParameterStage.java:680-688`).
  3. On completion, rebuild the combo model on the EDT, merge saved custom names case-insensitively, and preserve the user's current selection if it is still valid or was the pre-existing selected preset (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2303-2318`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2397-2404`).
  4. Guard action listeners during model replacement with an updating flag so `installFilterDescriptionUpdater` / `presetChanged` do not load macros just because options arrived (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2371-2378`, `src/main/java/flash/pipeline/ui/config/FilterParameterStage.java:680-688`).
- Invalidate or refresh the shared index only at explicit write points: after `saveCustomFilterPreset` writes a new preset (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2466-2474`), after embedded stage "Save preset" delegates to it (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:6251-6256`), and after named custom preset save (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:7423-7435`). There is no existing file-watch or directory-change event system to hook into; `FlashProjectLayout` only returns paths (`src/main/java/flash/pipeline/io/FlashProjectLayout.java:421-428`).
- Use one shared daemon worker for scan coalescing by project/bin root, but scope UI callbacks to the dialog/stage lifecycle. `ConfigQcDialog` calls `stage.onLeave` when navigating or closing (`src/main/java/flash/pipeline/ui/config/ConfigQcDialog.java:673-678`, `src/main/java/flash/pipeline/ui/config/ConfigQcDialog.java:687-693`), and `FilterParameterStage.onLeave` already cancels preview workers and clears UI state (`src/main/java/flash/pipeline/ui/config/FilterParameterStage.java:326-329`).

## Out Of Scope

- Caching macros across sessions to disk. Always re-scan per run or per invalidation; that avoids stale Dropbox state.
- Reorganising the filter stages or changing the macro editor UX.
- Making all `BinConfigIO.readFromDirectory` callers async. Many analysis, audit, and headless paths intentionally consume a fully populated `BinConfig` immediately (`src/main/java/flash/pipeline/audit/RunSettingsSnapshot.java:68`, `src/main/java/flash/pipeline/bin/BinSetupDispatcher.java:92`, `src/main/java/flash/pipeline/analyses/DeconvolutionAnalysis.java:1678`).
- Adding filesystem watchers. Use explicit invalidation at save points, not `WatchService`.

## Known Risks

- `FilterParameterStage` currently treats every constructor option except `Custom` as bundled (`src/main/java/flash/pipeline/ui/config/FilterParameterStage.java:206-212`, `src/main/java/flash/pipeline/ui/config/FilterParameterStage.java:1591-1593`). Async-added saved custom names must not make `readOnlyBase`, save-as state, or macro loading worse than today's behavior.
- Replacing a `JComboBox` model can fire listeners and load macros. Suppress updates while swapping and only fire user-visible behavior after the final selected item is restored (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2371-2378`, `src/main/java/flash/pipeline/ui/config/FilterParameterStage.java:680-688`).
- A user can press OK before the scan completes. Initial options must therefore be sufficient to preserve the current real preset, and the placeholder must be filtered out at read/save time (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2391-2394`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:552-553`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:1129-1131`).
- A scan can finish after the dialog/stage is gone. Use generation tokens and lifecycle checks; `ConfigQcDialog` calls `onLeave` on navigation/close (`src/main/java/flash/pipeline/ui/config/ConfigQcDialog.java:673-678`, `src/main/java/flash/pipeline/ui/config/ConfigQcDialog.java:687-693`) and `FilterParameterStage.onLeave` already clears state (`src/main/java/flash/pipeline/ui/config/FilterParameterStage.java:326-329`).
- There is no existing directory-change event mechanism to subscribe to. Invalidation must be explicit at save points (`src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:2466-2474`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:6251-6256`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:7423-7435`).
- Do not move all legacy inference onto async futures. CLI/headless and downstream analyses need immediate `BinConfig` values, while the UI dropdown only needs async option discovery (`src/main/java/flash/pipeline/bin/BinSetupDispatcher.java:92`, `src/main/java/flash/pipeline/cli/CLIArgumentParser.java:454-473`, `src/main/java/flash/pipeline/analyses/CreateBinFileAnalysis.java:912-938`).
- Do not leak threads. A shared named daemon executor is acceptable for coalescing slow folder scans, but UI callbacks must be scoped by generation/displayability rather than keeping dialog-specific executors alive (`src/main/java/flash/pipeline/ui/config/ConfigQcDialog.java:673-678`, `src/main/java/flash/pipeline/ui/config/FilterParameterStage.java:326-329`).
