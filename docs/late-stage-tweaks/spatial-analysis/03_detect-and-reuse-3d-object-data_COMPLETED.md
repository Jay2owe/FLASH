# Detect And Reuse 3D Object Data

## Why this stage exists

Spatial analysis should detect data already generated by 3D Object Analysis, avoid repeating that work, and explain the reuse in grey helper text.

## Prerequisites

- Read `src/main/java/flash/pipeline/analyses/SpatialAnalysis.java:172`.
- Read `src/main/java/flash/pipeline/analyses/SpatialAnalysis.java:317`.
- Read `src/main/java/flash/pipeline/analyses/SpatialAnalysis.java:1377`.
- Read `src/main/java/flash/pipeline/io/FlashProjectLayout.java:150`.

## Read first

- Spatial analysis already loads object CSV files from object-analysis output folders.
- CPC skip logic already checks whether CPC columns exist.
- The UI has help text for colocalisation/CPC, but it does not comprehensively explain which existing 3D object outputs will be reused.

## Scope

- Detect existing 3D object outputs relevant to spatial analysis before showing the dialog.
- Reuse existing columns/data for object counts, coloc/contact values, 3D morphology, and any process-length data when present.
- Disable or skip duplicate work where reuse is safe.
- Add grey helper text in the spatial dialog explaining what was detected and what will be reused.
- Log skip/reuse decisions so batch users see the same behavior.
- Add tests using object CSVs with and without existing columns.

## Out of scope

- Recomputing missing 3D object data inside spatial analysis.
- Changing 3D Object Analysis output schemas.

## Files touched

- `src/main/java/flash/pipeline/analyses/SpatialAnalysis.java`
- `src/main/java/flash/pipeline/io/FlashProjectLayout.java` only if read dirs miss current 3D object outputs
- `src/test/java/flash/pipeline/analyses/SpatialAnalysisTest.java`
- `src/test/java/flash/pipeline/analyses/SpatialAutoDetectTest.java`

## Implementation sketch

1. Add a small detection object that summarizes available columns/files from loaded 3D object outputs.
2. Populate it before building the spatial dialog.
3. Add grey helper text near Spatial Distances, Colocalization, and Morphometric Analysis sections.
4. Use the detection object to skip calculations already represented by existing columns.
5. Preserve existing CPC skip checks but route them through the same detection summary where possible.
6. Add tests for present CPC columns, present morphology columns, and absent data.

## Exit gate

- Spatial analysis explains detected 3D object data in the dialog.
- Duplicate calculations are skipped when equivalent data is already present.
- Missing data still runs when the user selects the option.
- Tests cover reuse and non-reuse paths.

## Known risks

- Column names may vary between old and new output layouts. Detection should support current names and important legacy names.
