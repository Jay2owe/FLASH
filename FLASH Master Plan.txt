# FLASH Master Plan: Advanced Feature Roadmap

This document outlines the long-term vision for taking the FLASH from a robust lab tool to a commercial-grade, highly publishable bio-image analysis software package.

## 1. Master Data Aggregation (The "Thesis Saver")
* **The Concept:** Automatically stitch hundreds of individual output CSVs into a single, analysis-ready master file.
* **Key Features:**
  * Runs automatically at the end of a batch process (3D Object or Intensity modules).
  * Parses standardized filenames (`Animal_Hemisphere_Region`) to create dedicated metadata columns.
  * Outputs `Project_Master_Objects.csv` and `Project_Master_Intensities.csv` ready for immediate import into Prism, R, or Python.

## 2. Automated QC Report Generation (Reproducibility)
* **The Concept:** Generate a lightweight `QC_Report.html` at the end of every run to guarantee scientific reproducibility and visual verification.
* **Key Features:**
  * Clean, formatted tables documenting the exact parameters used (thresholds, sizes, filter presets, headless state).
  * Timestamps, total processing time, and image counts.
  * **Visual Proof:** Embedded JPEG thumbnails showing the original image side-by-side with the `_objects` map for a random subset of images, allowing researchers (or reviewers) to verify segmentation quality at a glance.

## 3. CLI / PyImageJ Support (The Cluster Workflow)
* **The Concept:** Expose the pipeline's arguments so it can be triggered programmatically without ever opening the Fiji GUI.
* **Key Features:**
  * Allows execution via Bash scripts or Python (PyImageJ).
  * Enables high-performance computing (HPC) deployment where a user can loop through dozens of directories on a remote server.
  * Example invocation: `IJ.run("FLASH", "config_dir=[/path/] headless=true batch=true")`.

## 4. Deep Learning Integration (StarDist / Cellpose)
* **The Concept:** Elevate the pipeline from classical computer vision to state-of-the-art AI segmentation.
* **Key Features:**
  * Integrate the native ImageJ Java API for StarDist.
  * Provide a UI toggle to bypass standard auto-thresholding + watershed for complex channels (like dense DAPI/nuclei).
  * Drastically improves accuracy in tissue with highly overlapping cells.

## 5. Multi-threading / Parallel Processing
* **The Concept:** Process multiple massive Z-stacks simultaneously to slash batch processing times.
* **Key Features:**
  * Wrap the main image processing loops in a Java `ExecutorService`.
  * Capitalize on the newly implemented Headless architecture (which decoupled the pipeline from ImageJ's thread-unsafe `WindowManager`).
  * Utilizes the full power of modern multi-core processors (e.g., processing 4 to 8 images at the exact same time).
